### 1.`buildCommentTreeAsync` 方法（异步构建评论树）

```
private CompletableFuture<CommentTree> buildCommentTreeAsync(Long articleId, List<CommentDO> topComments) {
    return CompletableFuture.supplyAsync(() -> {
        Set<Long> topIds = topComments.stream().map(CommentDO::getId).collect(Collectors.toSet());
        List<CommentDO> subComments = commentDao.listSubCommentsRecursive(articleId, topIds);
        return buildTreeStructure(topComments, subComments);
    }, executor);
}

```

- 通过 **`CompletableFuture.supplyAsync`** 异步执行 `buildTreeStructure` 方法，避免阻塞主线程。
- 先获取 **所有顶级评论的 ID**（`topComments`）。
- 查询 **所有子评论**（通过 `commentDao.listSubCommentsRecursive(articleId, topIds)`）。
- 递归构建评论树，返回 `CommentTree` 结果。

### **📌 关键点**

- **异步执行**：使用 `executor` 线程池，提升性能。
- **数据库查询优化**：只查 **顶级评论的所有子评论**，减少 SQL 查询次数。

### 2. `buildTreeStructure` 方法（递归构建评论树）

```
private CommentTree buildTreeStructure(List<CommentDO> topComments, List<CommentDO> subComments) {
    Map<Long, TopCommentDTO> topMap = convertTopComments(topComments);
    Map<Long, SubCommentDTO> subMap = new HashMap<>();
    
    // 使用双队列处理层级关系
    Queue<CommentDO> processingQueue = new LinkedList<>(subComments);
    Set<Long> processedIds = new HashSet<>();

    while (!processingQueue.isEmpty()) {
        CommentDO comment = processingQueue.poll();
        if (processedIds.contains(comment.getId())) continue;

        SubCommentDTO sub = convertSubComment(comment);
        subMap.put(comment.getId(), sub);

        Long parentId = comment.getParentCommentId();
        if (parentId == null) {
            linkToTopComment(sub, topMap.get(comment.getTopCommentId()));
        } else {
            SubCommentDTO parent = subMap.get(parentId);
            if (parent != null) {
                parent.addChild(sub);
                sub.setParentContent(parent.getContent());
            } else {
                // 延迟处理依赖
                processingQueue.add(comment);
            }
        }
        processedIds.add(comment.getId());
    }

    return new CommentTree(
        topMap.values().stream()
            .sorted(Comparator.comparing(TopCommentDTO::getCreateTime).reversed())
            .toList()
    );
}

```

### **作用**

- 先将 **顶级评论** 转换成 `TopCommentDTO` 并存入 `topMap`。
- 遍历所有 **子评论**，找到其 **父评论** 并构建层级关系。
- 使用 **`Queue` 进行层级遍历**，支持 **无限层级的评论树**。
- **处理无法立即找到父级的子评论**（延迟处理）。

### **📌 关键点**

1. **转换 `CommentDO` 为 DTO**
   - `convertTopComments(topComments)` → 顶级评论 `TopCommentDTO`
   - `convertSubComment(comment)` → 子评论 `SubCommentDTO`
2. **双队列处理评论层级**
   - **`Queue processingQueue`** 存放子评论，逐层处理。
   - **`Set processedIds`** 记录已处理的评论，防止重复处理。
3. **关联评论**
   - 如果 **`parentId == null`**，说明是 **顶级评论的子评论**，直接加入 `TopCommentDTO`。
   - 否则找到 `parentId` 所对应的 `SubCommentDTO`，加入其 **子评论列表**。
4. **延迟处理子评论**
   - **如果父评论未找到**，说明父评论可能还未处理完成，将子评论重新加入 `Queue`，稍后处理。
5. **排序**
   - **按创建时间降序排列**，让最新的顶级评论 **排在前面**。

### 表结构设计

```
CREATE TABLE comment (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '评论ID',
    article_id BIGINT NOT NULL COMMENT '所属文章ID',
    
    user_id BIGINT NOT NULL COMMENT '评论用户ID',
    content TEXT NOT NULL COMMENT '评论内容',
    
    parent_comment_id BIGINT DEFAULT NULL COMMENT '直接父评论ID',
    top_comment_id BIGINT DEFAULT NULL COMMENT '顶级评论ID（根节点）',
    
    depth INT NOT NULL DEFAULT 1 COMMENT '评论层级深度',
    create_time DATETIME NOT NULL COMMENT '创建时间',
    
    INDEX idx_article (article_id),
    INDEX idx_parent (parent_comment_id),
    INDEX idx_top (top_comment_id),
    INDEX idx_time (create_time)
) ENGINE=InnoDB;
```
